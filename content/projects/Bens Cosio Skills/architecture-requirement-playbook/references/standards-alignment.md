# Standards Alignment Notes

Use these references to align architecture requirements with recognized practices:

- **ISO/IEC/IEEE 42010**: Architecture description principles
- **TOGAF-aligned governance**: Enterprise architecture process and decision governance
- **NIST CSF / OWASP ASVS**: Security and assurance controls
- **Cloud Well-Architected**: Reliability, security, performance, cost, operations
- **Privacy-by-design frameworks (e.g., GDPR-oriented controls)**: Data minimization, lawful processing, retention governance

## Practical Guidance

1. Reference standards in the document only when they are mapped to concrete controls.
2. For each control, capture the required evidence artifact and review gate.
3. Treat unmapped standards references as gaps, not compliance claims.
