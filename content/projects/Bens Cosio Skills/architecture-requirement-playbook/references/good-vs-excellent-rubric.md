# Good vs Excellent Requirements Rubric

Use this rubric to score requirement quality and prioritize improvements.

| Dimension | Good | Excellent |
|---|---|---|
| Clarity | Understandable prose | Unambiguous, structured, "shall" statements |
| Measurability | Some targets exist | All critical requirements have pass/fail criteria |
| Traceability | Partial links to objectives | Full objective → requirement → decision → test → metric chain |
| Governance | One-time review | Continuous governance with exception lifecycle |
| Risk Handling | Risks listed | Risks owned, time-bound, mitigated, and monitored |
| Operability | Monitoring mentioned | SLOs, alerts, runbooks, and readiness evidence defined |
| Compliance | Standards referenced | Controls mapped with explicit evidence artifacts |

## Suggested Scoring

- 0 = Missing
- 1 = Basic
- 2 = Adequate
- 3 = Strong
- 4 = Excellent

Prioritize remediation where score is 0-2 in high-risk domains (security, resilience, compliance, operations).
